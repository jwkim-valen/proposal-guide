# valen 제안서 제작 온보딩

> **Claude Code 지침:** 이 파일을 읽으면 아래 단계를 순서대로 진행한다.
> 각 단계의 "사용자 안내 메시지"를 그대로 출력하고, 사용자 응답 후 다음 단계로 이동.
> 단계를 건너뛰지 않는다. `guide/proposal_template.html`은 STEP 2 시점에만 다룬다.
> **주의 (중요):** 이 파일은 겉보기엔 2,200줄이지만 실제 용량은 2MB에 달한다 — 커버 이미지가 base64로 내장된 한 줄(약 1.9MB)이 있기 때문이다. **이 파일을 Read 도구로 통째로 읽지 않는다.** limit을 아무리 크게 줘도 그 1.9MB짜리 줄 하나가 그대로 컨텍스트에 로드되어 세션이 사실상 멈추거나 응답이 심각하게 느려진다. 대신 아래 STEP 2 "제작 절차"를 따른다: Bash `cp`로 파일 전체를 복제하고, 내용을 채울 위치만 좁은 범위로 Read → Edit 한다.

## STEP 2 — 제안서 제작

**Claude Code 작업 순서:**

```
1. Bash로 복제: cp guide/proposal_template.html {브랜드명}_proposal.html
   (Read로 통째로 불러오지 않는다 — 커버 이미지가 base64로 내장돼 있어 한 줄이 약 1.9MB.
    cp는 파일 내용을 보지 않고 복제만 하므로 안전하다.)
2. Edit이 가능하도록 새 파일을 한 번 Read — 단, 좁은 범위만: 예) Read(offset=1, limit=50)
   (이미지가 있는 줄은 절대 Read 범위에 포함하지 않는다. 어느 줄인지 모르면
    `grep -n "data:image" {브랜드명}_proposal.html | cut -c1-60`로 줄 번호만 먼저 확인하고 피해서 Read.)
3. 수집 변수를 변수 맵에 따라 각 위치에 적용 — 각 위치마다 Grep으로 줄 번호 확인 후 그 근처만 좁게 Read → Edit 반복 (전체 재작성 금지)
4. 섹션 수·순서 결정 (기본 13섹션, 내용에 따라 증감 가능)
5. 각 섹션 컴포넌트 선택 (TEMPLATE_GUIDE.md §7 판단 기준 참고) — §8 코드 최적화 원칙에 따라 새 CSS 규칙 추가 없이 기존 컴포넌트만 조합
6. 커버 이미지를 기본값과 다른 이미지로 바꿔야 하면: 이미지 파일을 base64로 변환(Bash `python3 -c "import base64,sys; print(base64.b64encode(open(sys.argv[1],'rb').read()).decode())" 경로`) 후, `cover-img-area`의 `src="data:image/png;base64,..."` 값만 교체. 기본 커버를 유지한다면 아무 작업도 필요 없음 (cp로 이미 따라옴).
   > **본문 섹션에 이미지를 추가할 때도 동일하게 적용한다.** 커버/로고뿐 아니라 어떤 이미지든 반드시 base64로 변환해 `src="data:image/...;base64,..."`로 내장한다. `src="assets/..."` 같은 상대경로나 사용자 로컬 파일 경로를 절대 그대로 참조하지 않는다 — 그렇게 하면 파일을 `archive/`로 옮기거나, 다른 사람에게 전달하거나, Netlify 등에 HTML 파일 하나만 배포할 때 이미지 파일이 함께 있지 않아 깨진다.
7. 루트에 {브랜드명}_proposal.html 생성 완료
8. archive/{오늘날짜}_{브랜드명}/ 폴더 생성 후 동일 파일 복사
9. 핵심 기능 무결성 확인 — 아래 체크리스트의 "핵심 함수" 항목 grep 확인 (grep -c는 개수만 반환하므로 안전)
```

**적용 체크리스트:**

- [ ] `<title>` → `{브랜드명} 제안 — valen`
- [ ] `const EDIT_MODE = false;` — 읽기 전용으로 생성
- [ ] `.intro-eye` → `{브랜드명} × VALEN · {제안유형}`
- [ ] `.intro-h1` → `{핵심메시지}` + `<span>{서브메시지}</span>`
- [ ] `.intro-sub` → `{한줄설명}`
- [ ] `.intro-tagline` → `{태그라인_EN}` / `{태그라인_KO}`
- [ ] `.intro-kpi` × 3 → `{수치}{단위}` + `{설명}`
- [ ] `.intro-desc` → 제안 배경 2–3줄
- [ ] `nav.toc` 항목 → 섹션 수·순서에 맞게 정비
- [ ] 각 `section` `.s-meta` / `h2` / `.lead` / `.s-body` 내용 채우기
- [ ] 커버 이미지 — 기본 유지 시 확인 불필요(base64 내장, cp로 자동 포함), 교체 시 base64 변환 후 `data:image/png;base64,...` 치환 확인
- [ ] 본문에 추가한 이미지 전체 — 예외 없이 base64 내장 확인 (`grep -c 'src="data:' {파일}`이 `<img` 태그 수와 일치해야 함). `src="assets/` 등 상대경로가 하나라도 남아있으면 배포·전달 시 깨지므로 반드시 base64로 교체
- [ ] 푸터 로고 — 기본 유지 (변경 요청 없는 한 그대로 둠)
- [ ] `footer span` → `{브랜드명} 제안 (KR v1) · Confidential`
- [ ] 파일명 → `{브랜드명}_proposal.html`
- [ ] 아카이브 → `archive/{YYYYMMDD}_{브랜드명}/{브랜드명}_proposal.html`
- [ ] **핵심 함수** — 아래 함수가 생성된 파일에 모두 존재하는지 grep 확인 (하나라도 없으면 template Read가 잘린 것 → 다시 Read):
      `downloadPNG` · `downloadPDF` · `togglePngPanel` · `_loadHtml2Canvas` · `_loadJsPDF` · `enterPresent` · `exitPresent` · `saveEdits` · `restoreEdits` · `downloadHTML` · `initEditMode`

**완료 후 사용자 안내 메시지:**

## 자주 묻는 질문

**Q. 브라우저 새로고침 후 수정 내용이 사라졌어요.**
A. localStorage에 저장된 데이터입니다. 같은 브라우저·같은 경로로 열면 복원됩니다. 다른 브라우저나 시크릿 모드에서는 보이지 않습니다.

**Q. "HTML 저장" 버튼과 "수정 완료" 입력의 차이는?**
A. "HTML 저장" → 브라우저 localStorage에 저장 후 읽기 전용 전환. 실제 HTML 파일은 변경되지 않습니다. "수정 완료" → Claude Code가 `EDIT_MODE` 변수를 false로 수정. 내용 변경 없이 편집 바만 숨깁니다.

**Q. 실제 HTML 파일에 수정 내용을 영구 반영하려면?**
A. 현재는 localStorage 기반 임시 저장입니다. HTML 파일에 직접 반영이 필요하면 Claude Code에 수정 내용을 전달해 주세요.

**Q. 섹션 수를 늘리거나 줄이고 싶어요.**
A. 기본 13섹션이지만 자유롭게 조정 가능합니다. 섹션 구성안을 알려주시면 반영합니다. TOC와 섹션 `id`는 항상 일치해야 합니다.

**Q. 이미지/PDF 저장 같은 기능이 특정 제안서에서만 안 눌려요.**
A. 대부분 `guide/proposal_template.html`을 Read로 통째로 불러와 복사하려다 생긴 문제입니다 (커버 이미지 base64 한 줄이 약 1.9MB라 Read 자체가 잘리거나 매우 느려짐). STEP 2처럼 Bash `cp`로 파일을 복제했는지, 핵심 함수를 재구성이 아니라 부분 Read+Edit로 채웠는지 확인하세요. STEP 2의 "핵심 함수" 체크리스트로 누락 여부를 확인하고, 누락됐다면 template에서 해당 함수 블록만 좁은 범위로 Read하여 제안서 파일에 추가하세요. 절대 누락된 상태로 기능 자체를 삭제하거나 버튼만 숨기지 말 것.

**Q. 본문에 추가한 이미지가 내 컴퓨터에선 잘 보이는데 다른 사람 컴퓨터나 Netlify 같은 배포 환경에서는 깨져요.**
A. 이미지가 base64로 내장되지 않고 `src="assets/..."` 같은 상대경로나 로컬 파일 경로로 참조된 경우입니다. 내 컴퓨터에서는 같은 폴더에 이미지 파일이 실제로 있어서 보이지만, HTML 파일 하나만 다른 사람에게 전달하거나 Netlify Drop 등에 올리면 그 이미지 파일은 함께 이동하지 않아 깨집니다. `grep -c 'src="data:' {파일}`로 나온 개수가 `<img` 태그 총 개수와 다르면 상대경로가 남아있는 것이니, 해당 이미지를 base64로 변환해 `src="data:image/...;base64,..."`로 교체하세요 (STEP 2 항목 6 참고).

**Q. 제안서나 템플릿을 계속 수정하다 보니 파일이 너무 길어져서 Claude Code가 못 불러와요. 어떻게 정리하나요?**
A. 예방이 우선입니다 — 제작 시점부터 STEP 2 체크리스트로 핵심 함수 무결성을 확인하면 잘린 채로 넘어가는 일 자체를 막을 수 있습니다.
이미 파일이 길어져서 문제가 생겼다면, 위 "핵심 함수" 목록에 있는 기능은 절대 건드리지 않는다는 전제로 다음만 정리 대상으로 삼으세요:
  - 더 이상 어떤 버튼/이벤트에서도 호출되지 않는 죽은 함수·CSS (grep으로 호출부가 0건인지 먼저 확인)
  - 중복된 스타일 블록
정리 후에는 반드시 "핵심 함수" grep 체크리스트를 다시 돌려 전부 남아있는지 확인합니다. 확신 없는 코드는 삭제하지 말고 그대로 둡니다.
