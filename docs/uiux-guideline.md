# UI/UX Visual Hierarchy Guideline

## Purpose

이 문서는 웹 및 앱 인터페이스, 특히 HTML 제안서 템플릿 제작 과정에서 시각적 계층을 명확하게 설계하기 위한 기준 문서입니다.

좋은 시각적 계층은 사용자가 화면을 보았을 때 무엇을 먼저 보고, 어떤 순서로 이해하며, 다음에 어떤 행동을 해야 하는지 자연스럽게 안내합니다.

핵심 기준은 다음과 같습니다.

- 사용자가 가장 중요한 정보를 먼저 보게 한다.
- 정보의 중요도에 따라 크기, 대비, 여백, 위치를 조절한다.
- 콘텐츠 흐름이 이야기처럼 자연스럽게 이어지도록 구성한다.
- 디자인 장식보다 정보 전달과 사용자의 이해를 우선한다.
- 같은 역할의 요소는 같은 스타일로 반복한다.

---

## 1. One Primary Action per Page

한 화면 또는 한 섹션에는 가장 중요한 행동 또는 메시지를 하나만 두는 것이 좋습니다.

사용자가 다음에 무엇을 해야 할지 추측하게 만들면 안 됩니다.

### Apply

- 한 섹션에는 하나의 핵심 메시지만 둔다.
- CTA가 필요한 경우 가장 중요한 CTA 하나를 우선한다.
- 보조 CTA는 시각적 강도를 낮춘다.
- 사용자가 다음에 봐야 할 정보가 자연스럽게 이어지도록 배치한다.

### Avoid

- 한 화면에 여러 개의 강조 버튼을 배치하는 것
- 모든 요소를 동일하게 강조하는 것
- 사용자가 어디를 클릭해야 할지 모르게 만드는 것

---

## 2. Use Generous White Space

여백은 비어 있는 공간이 아니라 정보의 관계를 정리하는 디자인 요소입니다.

넉넉한 여백은 화면에 숨 쉴 공간을 만들고, 사용자가 정보를 더 쉽게 이해하도록 돕습니다.

### Apply

- 관련 있는 정보는 가깝게 배치한다.
- 관련 없는 정보는 충분히 떨어뜨린다.
- 섹션과 섹션 사이에는 큰 여백을 둔다.
- 카드 내부 여백과 카드 간 간격을 구분한다.
- 핵심 메시지 주변에는 여백을 넓게 둔다.

### Avoid

- 모든 정보를 한 화면에 과도하게 밀어 넣는 것
- 제목과 본문 간격이 불분명한 것
- 카드 내부 여백과 외부 여백이 같은 것
- 여백 없이 선과 박스로만 영역을 나누는 것

---

## 3. Typography Pyramid

타이포그래피는 시각적 계층을 만드는 가장 중요한 도구입니다.

H1, H2, H3, Body, Caption의 역할이 명확해야 사용자가 정보의 중요도를 빠르게 이해할 수 있습니다.

### Basic Structure

```txt
H1: 페이지 또는 섹션의 가장 중요한 메시지
H2: 주요 섹션 제목
H3: 카드 또는 하위 그룹 제목
Body: 상세 설명
Caption: 보조 정보, 출처, 주석
```

### Apply

- 제목, 요약, 본문, 보조 정보의 크기 차이를 명확히 둔다.
- 제목은 짧고 선명하게 작성한다.
- 본문은 충분한 행간을 유지한다.
- 같은 역할의 텍스트는 같은 크기와 굵기를 사용한다.
- 중요한 문장만 굵기 또는 컬러로 강조한다.

### Avoid

- 모든 텍스트를 크게 만드는 것
- 같은 섹션 안에서 제목 스타일이 계속 바뀌는 것
- 한 문장 안에서 과도한 크기 차이를 주는 것
- 장식용 서체를 무분별하게 섞는 것

---

## 4. Easy Page Flow

좋은 레이아웃은 사용자를 자연스럽게 다음 내용으로 안내합니다.

화면은 단순히 정보가 놓인 공간이 아니라, 사용자가 읽어 내려가는 이야기 구조여야 합니다.

### Recommended Flow

```txt
문제 제기
→ 인사이트
→ 전략
→ 실행 방안
→ 기대 효과
→ 일정 / 범위
→ 마무리
```

### Section Flow

```txt
큰 제목
→ 짧은 요약
→ 핵심 카드 또는 시각 자료
→ 상세 설명
→ 다음 섹션으로 연결
```

### Apply

- 각 섹션의 시작 부분에 요약 문장을 둔다.
- 사용자가 훑어봐도 흐름을 이해할 수 있게 구성한다.
- 긴 문단보다 짧은 그룹 단위로 정보를 나눈다.
- 섹션 간 연결이 어색하지 않도록 순서를 설계한다.

### Avoid

- 긴 문단이 반복되는 구조
- 제목 없이 표나 카드가 바로 등장하는 구조
- 정보 흐름이 앞뒤로 반복되는 구조
- 중요한 결론이 너무 뒤에 숨겨지는 구조

---

## 5. Size Hierarchy

사용자는 보통 큰 요소를 먼저 봅니다.

따라서 가장 중요한 메시지는 다른 요소보다 명확히 커야 합니다.

### Example

```txt
가장 큼: 핵심 제안 메시지
중간: 섹션 제목
작음: 설명 본문
더 작음: 캡션, 보조 정보
```

### Apply

- 중요한 메시지에만 큰 크기를 사용한다.
- 섹션 제목과 카드 제목의 크기를 구분한다.
- 본문 크기는 읽기 편한 수준으로 유지한다.
- 보조 정보는 작게 하되 읽을 수 있어야 한다.

### Avoid

- 모든 제목을 같은 크기로 사용하는 것
- 중요하지 않은 요소가 가장 크게 보이는 것
- 본문이 지나치게 작아지는 것
- 카드 안의 정보 위계가 없는 것

---

## 6. Contrast

시각적 계층은 크기뿐 아니라 대비로도 만들어집니다.

대비는 사용자의 시선을 특정 요소로 유도하는 강력한 방법입니다.

### Types of Contrast

```txt
밝음 / 어두움
큰 글자 / 작은 글자
굵은 글자 / 얇은 글자
컬러 / 무채색
채워진 버튼 / 라인 버튼
넓은 여백 / 촘촘한 요소
정적인 요소 / 움직이는 요소
```

### Apply

- 가장 중요한 요소에만 강한 대비를 준다.
- 버튼과 일반 텍스트의 차이를 명확히 한다.
- 활성 상태와 비활성 상태를 구분한다.
- 배경과 텍스트의 대비를 충분히 확보한다.

### Avoid

- 모든 요소에 강조색을 사용하는 것
- 대비가 약해 텍스트가 읽히지 않는 것
- 중요한 요소와 보조 요소가 같은 강도로 보이는 것
- 색상만으로 상태를 구분하는 것

---

## 7. Alignment and Grid

정렬은 디자인을 안정적으로 보이게 하고 정보의 관계를 명확히 합니다.

정렬이 흐트러지면 화면이 지저분해 보일 뿐 아니라, 사용자가 정보 구조를 이해하기 어려워집니다.

### Apply

- 하나의 섹션 안에서는 정렬 기준을 유지한다.
- 제목, 본문, 카드, 이미지의 기준선을 맞춘다.
- 좌측 정렬을 기본으로 하되, 의도 있는 경우에만 중앙 정렬을 사용한다.
- 카드 그리드는 동일한 간격과 높이 기준을 유지한다.

### Avoid

- 제목은 왼쪽, 본문은 중앙, 버튼은 오른쪽처럼 기준이 섞이는 것
- 카드 간 간격이 불규칙한 것
- 이미지와 텍스트의 기준선이 맞지 않는 것
- 여백과 정렬이 섹션마다 달라지는 것

---

## 8. Grouping

사람은 가까이 있는 요소를 하나의 그룹으로 인식합니다.

관련 있는 정보는 가까이, 관련 없는 정보는 멀리 배치해야 합니다.

### Card Example

```txt
카드 제목
카드 설명
보조 정보
```

위 요소들은 서로 가까워야 합니다.  
반면 다른 카드와는 충분히 떨어져 있어야 합니다.

### Apply

- 제목과 설명은 하나의 그룹처럼 보이게 배치한다.
- 카드끼리는 명확히 분리한다.
- 같은 주제의 정보는 하나의 섹션 또는 카드 안에 묶는다.
- 표, 그래프, 설명은 서로의 관계가 보이도록 배치한다.

### Avoid

- 어떤 설명이 어떤 제목에 속하는지 불분명한 것
- 관련 없는 요소가 너무 가까운 것
- 관련 있는 요소 사이에 불필요한 여백이 큰 것
- 그룹 구분을 선이나 박스에만 의존하는 것

---

## 9. Repetition and Consistency

좋은 UI는 같은 역할의 요소가 같은 모습으로 반복됩니다.

사용자는 반복되는 패턴을 학습하고, 다음 화면에서도 같은 방식으로 정보를 이해합니다.

### Apply

- 모든 섹션 제목은 같은 스타일을 사용한다.
- 모든 버튼은 같은 높이, 라운드, 굵기를 유지한다.
- 모든 카드 제목은 같은 크기와 굵기를 사용한다.
- 표, 리스트, 캡션 스타일을 통일한다.
- 인터랙션 상태도 일관되게 유지한다.

### Avoid

- 섹션마다 제목 스타일이 달라지는 것
- 버튼마다 크기와 색상이 다른 것
- 카드마다 내부 여백이 다른 것
- 같은 기능의 UI가 다른 방식으로 작동하는 것

---

## 10. Scannable Structure

사용자는 화면을 처음부터 끝까지 꼼꼼히 읽기보다 먼저 훑어봅니다.

좋은 UI는 대충 훑어봐도 구조와 핵심 메시지가 보여야 합니다.

### Apply

- 큰 제목과 짧은 요약을 먼저 보여준다.
- 긴 문단은 카드, 리스트, 표로 나눈다.
- 핵심 키워드는 제목 또는 라벨로 드러낸다.
- 한 섹션에서 말하고자 하는 바를 첫 화면에서 이해할 수 있게 한다.

### Avoid

- 긴 문단만 반복되는 구조
- 제목 없이 내용이 이어지는 구조
- 핵심 메시지가 본문 중간에 숨겨지는 구조
- 모든 정보가 같은 강도로 나열되는 구조

---

## 11. Information Density

좋은 디자인은 정보가 적은 디자인이 아니라, 정보 밀도가 잘 조절된 디자인입니다.

정보가 많은 영역일수록 장식보다 구조가 중요합니다.

### Apply

- 핵심 메시지 섹션은 여백을 넓게 둔다.
- 전략 카드 섹션은 카드 간격을 균일하게 둔다.
- 일정표와 견적표는 행과 열 기준을 명확히 한다.
- 숫자는 정렬 기준을 통일한다.
- 복잡한 정보는 요약과 상세로 나눈다.

### Avoid

- 정보가 많은 표에 과한 장식을 넣는 것
- 핵심 메시지와 상세 정보가 같은 밀도로 보이는 것
- 너무 많은 카드가 한 화면에 몰리는 것
- 여백 없이 텍스트와 숫자가 촘촘히 배치되는 것

---

## 12. Eye Flow

사용자의 시선은 보통 다음 요소를 먼저 봅니다.

```txt
큰 것
→ 진한 것
→ 색이 있는 것
→ 움직이는 것
→ 가까운 것
```

웹에서는 대체로 왼쪽 위에서 오른쪽 아래로 읽는 흐름이 많습니다.

### Apply

- 핵심 메시지는 화면의 시작 지점에 둔다.
- 다음 행동은 읽기 흐름의 끝에 둔다.
- 중요한 카드나 숫자는 시선 흐름 안에 배치한다.
- 움직임이 있는 요소는 꼭 필요한 곳에만 사용한다.

### Avoid

- 시선이 여러 방향으로 흩어지는 구조
- 중요한 정보가 화면 가장자리나 하단에 숨는 구조
- 움직이는 요소가 본문 읽기를 방해하는 것
- 장식 요소가 핵심 메시지보다 먼저 보이는 것

---

## 13. Color Role System

색상은 예쁘게 쓰는 것이 아니라 역할을 나누어 사용해야 합니다.

### General Role

```txt
Primary color: 브랜드 인상
Accent color: 강조 / active / CTA
Gray: 본문 / 배경 / 구분선
Red: 오류 / 경고
Green: 성공 / 완료
Blue: 정보 / 신뢰
```

### Valenlife Proposal Template Example

```txt
Core Blue: 브랜드 무드, 표지, 주요 섹션
Origin Green: 강조, active, 포인트
White / Gray: 본문과 정보 정리
Advertiser Color: 필요한 경우 제한적 accent
```

### Apply

- 강한 컬러는 적게 사용한다.
- 브랜드 컬러와 기능 컬러의 역할을 구분한다.
- 강조 컬러는 active, tag, button, key number 등에 제한적으로 사용한다.
- 광고주 브랜드 컬러를 사용할 경우 전체 시스템을 깨지 않게 일부 accent에만 반영한다.

### Avoid

- 세 가지 이상의 주요 컬러를 동시에 사용하는 것
- 모든 카드에 다른 색상을 사용하는 것
- 배경색과 텍스트 대비가 약한 조합
- 브랜드 컬러를 너무 넓게 반복 사용하는 것

---

## 14. Clear CTA

CTA는 사용자가 다음에 해야 할 행동입니다.

CTA가 많으면 사용자는 오히려 행동하지 않습니다.

### Apply

- 한 화면에서 가장 중요한 CTA 하나를 우선한다.
- 버튼 문구는 구체적으로 작성한다.
- 버튼 주변에 충분한 여백을 둔다.
- 클릭 가능한 형태가 명확해야 한다.

### Good CTA Copy

```txt
제안 내용 검토하기
프로젝트 일정 확인하기
다음 단계 논의하기
문의하기
```

### Weak CTA Copy

```txt
확인
바로가기
더보기
클릭
```

### Avoid

- 같은 강도의 CTA가 여러 개 있는 것
- 버튼처럼 보이지 않는 CTA
- 버튼 문구가 모호한 것
- CTA가 본문 사이에 묻히는 것

---

## 15. State Clarity

UI에서는 현재 상태가 명확해야 합니다.

사용자는 내가 어디에 있는지, 무엇이 선택되었는지, 무엇을 누를 수 있는지 바로 알아야 합니다.

### Common States

```txt
Default
Hover
Active
Selected
Disabled
Error
Success
Loading
```

### Apply

- 좌측 목차에는 현재 섹션 active 상태를 표시한다.
- 버튼 hover 상태를 명확히 한다.
- 선택된 필터나 탭은 다른 상태와 구분한다.
- 클릭 불가능한 요소는 disabled 상태를 명확히 한다.

### Avoid

- active와 hover가 구분되지 않는 것
- 클릭 가능한 요소와 일반 텍스트가 비슷한 것
- 현재 위치를 알 수 없는 내비게이션
- 색상만으로 상태를 구분하는 것

---

## 16. Responsive Hierarchy

PC에서 좋은 디자인이 모바일에서 무너지면 좋은 디자인이 아닙니다.

반응형에서도 정보의 중요도 순서가 유지되어야 합니다.

### Common Responsive Changes

```txt
좌우 2열 → 세로 1열
좌측 목차 → 상단 요약 또는 숨김
큰 타이틀 → 적절히 축소
넓은 여백 → 적절히 압축
표 → 카드형 또는 가로 스크롤
```

### Apply

- 모바일에서도 제목, 요약, 본문 순서를 유지한다.
- 카드 그리드는 1열로 자연스럽게 전환한다.
- 좌측 목차는 숨기거나 상단 요약형으로 전환한다.
- 표는 가로 스크롤 또는 카드형으로 바꾼다.
- 터치 영역은 충분히 크게 유지한다.

### Avoid

- PC 레이아웃을 그대로 축소하는 것
- 모바일에서 본문이 너무 작아지는 것
- 목차가 화면을 과도하게 차지하는 것
- 카드가 너무 좁아져 읽기 어려운 것

---

## 17. Accessibility and Readability

디자인이 좋아 보여도 읽기 어렵다면 좋은 UI가 아닙니다.

### Checkpoints

```txt
본문 글자가 너무 작지 않은가?
행간이 충분한가?
텍스트와 배경 대비가 충분한가?
클릭 영역이 너무 작지 않은가?
색상만으로 정보를 구분하고 있지 않은가?
긴 문단이 너무 많지 않은가?
```

### Apply

- 본문은 최소 14px 이상을 권장한다.
- 중요한 본문은 16px 이상을 사용한다.
- 줄 간격은 텍스트 크기의 1.4~1.6배를 기준으로 한다.
- 배경 위 텍스트 대비를 충분히 확보한다.
- 색상 외에도 굵기, 아이콘, 위치 등으로 상태를 함께 표시한다.

### Avoid

- 밝은 배경 위에 연한 회색 텍스트를 사용하는 것
- 긴 문장을 작은 글씨로 배치하는 것
- 클릭 영역이 너무 작은 것
- 중요한 정보를 색상 하나로만 표시하는 것

---

## 18. Remove Unnecessary Elements

좋은 디자인에는 불필요한 요소가 적습니다.

모든 요소에는 이유가 있어야 합니다.

### Ask Before Adding

```txt
이 선은 왜 필요한가?
이 배경색은 왜 들어갔는가?
이 아이콘은 이해를 돕는가?
이 애니메이션은 읽기를 돕는가?
이 카드는 정말 카드여야 하는가?
이 섹션은 제안 흐름에 필요한가?
```

### Apply

- 의미 없는 장식은 제거한다.
- 정보 구분이 여백으로 가능하다면 선을 줄인다.
- 아이콘은 이해를 도울 때만 사용한다.
- 애니메이션은 읽기 흐름을 도울 때만 사용한다.

### Avoid

- 단순 장식을 위한 선, 박스, 배경 반복
- 모든 카드에 아이콘을 넣는 것
- 이유 없는 그라디언트와 그림자
- 내용보다 효과가 먼저 보이는 디자인

---

## 19. Proposal Template-Specific Principles

HTML 제안서 템플릿에서는 일반 UI보다 정보 전달과 설득 흐름이 더 중요합니다.

### Core Principles

```txt
1. 한 섹션에는 하나의 핵심 메시지만 둔다.
2. 제목, 요약, 본문, 보조 정보의 크기 차이를 명확히 둔다.
3. 여백을 충분히 사용해 정보 그룹을 구분한다.
4. Core Blue는 브랜드 인상, Origin Green은 강조에만 사용한다.
5. 카드, 표, 목차, 버튼 스타일을 반복적으로 유지한다.
6. 좌측 목차 active 상태로 현재 위치를 알려준다.
7. 스크롤 인터랙션은 읽기를 돕는 수준으로만 사용한다.
8. 제안서 내용이 바뀌어도 깨지지 않는 유연한 구조를 만든다.
9. 광고주 브랜드 컬러는 제한적으로만 반영한다.
10. 디자인보다 정보 전달과 설득 흐름을 우선한다.
```

### Recommended Proposal Section Pattern

```txt
Section Label
→ Section Title
→ One-line Summary
→ Key Content Block
→ Supporting Detail
→ Transition to Next Section
```

### Interaction Level

```txt
권장:
- Section fade-up
- Card sequential reveal
- Sticky TOC active state
- Smooth scroll
- Subtle hover

비권장:
- Strong parallax
- Heavy gradient animation
- Complex scroll pinning
- Excessive glow
- Full-screen transition
```

## 20. Final Principle

좋은 UI/UX의 시각적 계층은 예쁘게 꾸미는 것이 아니라, 사용자가 정보를 헷갈리지 않고 순서대로 이해하게 만드는 설계입니다.

디자인 검토 시 항상 다음 질문을 기준으로 판단합니다.

```txt
무엇이 가장 먼저 보이는가?
사용자는 이 섹션의 목적을 바로 이해하는가?
다음에 어디를 봐야 하는지 자연스럽게 이어지는가?
중요하지 않은 요소가 너무 튀지 않는가?
정보의 관계가 여백과 정렬로 명확히 드러나는가?
이 디자인은 콘텐츠가 바뀌어도 유지될 수 있는가?
```
