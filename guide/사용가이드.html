이 시스템은 Claude Code로 제안서를 자동 생성하고, 브라우저에서 텍스트를 직접 수정한 뒤 단일 HTML 파일로 저장하는 워크플로우입니다. 외부 툴 없이 브라우저 하나로 완성된 제안서를 전달할 수 있습니다.

01 · 시스템 소개

Claude Code로 자동 생성하고
브라우저에서 직접 편집한다

브랜드 정보를 입력하면 valen 디자인 시스템이 적용된 제안서가 자동으로 만들어집니다. 이후 편집과 저장까지 브라우저 하나로 처리합니다.

STEP 1
자동 제작

Claude Code에 /온보딩 입력 → 브랜드 정보 입력 → 제안서 HTML 자동 생성. 컴포넌트·컬러·타이포 모두 valen 가이드라인 기반.

STEP 2
인라인 편집

브라우저에서 직접 텍스트 클릭 수정. 수정 내용은 localStorage에 자동 저장되어 브라우저를 닫아도 유지됩니다.

STEP 3
파일 저장 · 공유

"HTML 저장" 클릭 → OS 다이얼로그 → 수정본 폴더에 저장. 저장된 파일은 레이아웃과 컬러가 원본 그대로 유지됩니다.

핵심

제안서는 단일 .html 파일 하나로 완결됩니다. 외부 서버, 라이브러리, 의존성 없이 브라우저에서 바로 열 수 있고, 이메일 첨부나 공유 폴더로 전달이 가능합니다.

02 · 제안서 만들기

5단계로 제안서 완성

Claude Code 터미널에서 명령어를 입력하고, 브랜드 정보를 제공하면 자동으로 제안서가 생성됩니다.

1
Claude Code 열기

제안서_Guide 폴더에서 Claude Code 실행. 세션 시작 시 가이드라인 파일들을 자동으로 읽습니다.

2
/온보딩 입력

채팅창에 /온보딩을 입력하면 브랜드 정보 수집을 시작합니다.

3
브랜드 정보 입력

브랜드명, 산업, 타겟, 핵심 메시지 등을 질문에 따라 입력합니다.

4
제안서 생성 완료

루트 폴더에 {브랜드명}_proposal.html이 자동 생성됩니다. 브라우저로 바로 열어 확인합니다.

5
아카이브 저장

archive/{날짜}_{브랜드명}/ 폴더에 복사 저장합니다. 이후 편집은 이 아카이브 파일에서 진행합니다.

팁

제안서가 생성된 후 내용을 크게 수정하고 싶다면 Claude Code에서 직접 요청합니다. 텍스트 수정 수준의 변경은 편집 모드에서 브라우저로 처리합니다.

03 · 편집하기

브라우저에서 텍스트를 직접 수정한다

편집 모드를 켜면 모든 텍스트 요소를 클릭해서 바로 수정할 수 있습니다. 수정 내용은 자동으로 저장됩니다.

1
편집 모드 ON

Claude Code에 "편집 모드"라고 입력합니다. EDIT_MODE = true로 전환됩니다.

2
브라우저 새로고침

Cmd+R로 새로고침하면 상단에 편집 바가 나타납니다. 초록 점이 깜빡이면 편집 모드 활성화 상태입니다.

3
텍스트 클릭 · 수정

수정할 텍스트를 클릭합니다. 초록 테두리가 표시되면 타이핑으로 수정합니다. 다른 곳 클릭 시 자동 저장됩니다.

4
편집 완료

Claude Code에 "편집 완료"를 입력하거나, "HTML 저장" 버튼을 클릭해 파일로 저장합니다.

자동 저장

수정 내용은 브라우저 localStorage에 자동 저장됩니다. 브라우저를 닫았다가 다시 열어도 수정 내용이 유지됩니다. 단, 다른 브라우저나 시크릿 탭에서는 저장된 내용이 보이지 않습니다.

주의

이미지, 링크, 로고는 편집 모드에서 수정할 수 없습니다. 이미지 교체는 assets/ 폴더의 파일을 직접 교체하거나 Claude Code에 요청합니다.

04 · HTML 저장

수정 완료 후 파일로 저장하기

"HTML 저장" 버튼을 누르면 편집 내용이 반영된 깨끗한 최종 파일이 생성됩니다. 레이아웃과 컬러가 원본 그대로 유지됩니다.

1
HTML 저장 클릭

편집 바 오른쪽의 초록 "HTML 저장" 버튼을 클릭합니다.

2
저장 위치 선택

OS 기본 저장 다이얼로그가 열립니다. archive/{날짜}_{브랜드명}/수정본/ 폴더로 이동해 저장합니다.

3
저장 완료

파일명은 {브랜드명}_proposal_{날짜}.html 형태로 자동 제안됩니다. 그대로 저장하거나 원하는 이름으로 변경합니다.

저장된 파일의 특성

저장된 파일은 편집 모드 없이 열립니다. 레이아웃, 컬러, 모든 구조가 원본 그대로 보존됩니다. 클라이언트에게 전달하거나 PDF 변환 시 이 파일을 사용합니다.

수정본 폴더는 git에서 제외(gitignore)되어, 각자의 파일이 다른 팀원의 작업에 영향을 주지 않습니다.

05 · 파일 관리

폴더 구조와 버전 관리

제안서 파일은 아카이브 폴더에 날짜별로 정리됩니다. 수정본은 별도 폴더에 저장되어 git 추적에서 분리됩니다.

작업용 파일루트에 위치, 편집 워크플로우용
  • {브랜드명}_proposal.html — 작업 중인 원본
  • assets/ — 커버 이미지, 로고
  • guide/ — 템플릿 · 가이드 문서
아카이브버전 백업 · 수정본 저장
  • archive/{날짜}_{브랜드명}/ — 버전 폴더
  • {브랜드명}_proposal.html — 아카이브 원본
  • 수정본/ — HTML 저장 파일 (gitignore)
Git 동기화

원본 제안서 파일과 가이드 문서는 git으로 팀 전체가 동기화합니다. 수정본 폴더는 gitignore 처리되어 각자의 편집 파일이 충돌 없이 관리됩니다. 새 제안서를 만든 후 git push하면 팀원들이 pull로 받을 수 있습니다.

06 · 명령어 레퍼런스

전체 명령어 한눈에

Claude Code 채팅창에 아래 명령어를 입력합니다.

명령어 동작 사용 시점
/온보딩 브랜드 정보 수집 → 제안서 자동 생성 새 제안서 처음 만들 때
/새제안서 정보 수집 단계부터 재시작 다시 처음부터 만들 때
편집 모드 EDIT_MODE = true 전환 → 브라우저 편집 활성화 텍스트 수정 전
편집 완료 EDIT_MODE = false 전환 → 원본 파일 정리 편집 마친 후
브라우저 액션

아래는 브라우저 편집 바에서 직접 클릭하는 버튼입니다. Claude Code 명령어가 아닙니다.

HTML 저장 — 수정 내용이 반영된 최종 파일을 저장합니다.
초기화 — localStorage의 모든 수정 내용을 삭제하고 원본으로 되돌립니다.
